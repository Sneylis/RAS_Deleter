# RAS Deleter v2.0.0

Утилита для полного удаления популярного Remote Access Software (RAS) с Windows-хостов.

## Описание

**RAS Deleter** — это Rust-приложение, разработанное для удаления популярных инструментов удалённого доступа (RAS) с Windows-систем. Утилита выполняет глубокое сканирование и многоэтапное удаление, обеспечивая полное избавление от поддерживаемых RAS-инструментов без остатка.

Программа поставляется в двух форматах:
- **ras_del.exe** — основной исполняемый файл
- **RAST_Deleter.exe** — самораспаковывающийся архив, который автоматически запускает `ras_del.exe --silence -log-dir "%LOCALAPPDATA%\"`

## Ключевые возможности

✓ **Многоуровневое сканирование** — находит RAS независимо от пути установки
✓ **Динамическое разрешение путей** — реестр HKLM/HKCU + активные процессы + стандартные директории
✓ **Завершение процессов** — graceful stop (2 сек) → force kill
✓ **Встроенный деинсталлятор** — безопасный прямой вызов exe без ошибок cmd /C
✓ **Удаление служб Windows** — через `sc delete` с проверкой статуса
✓ **Очистка файлов** — удаление директорий с fallback на `cmd /c rd /s /q` при ошибках доступа
✓ **Очистка реестра** — удаление ключей HKLM\Software\Microsoft\Windows\CurrentVersion\Uninstall
✓ **Очистка файрвола** — удаление правил `netsh advfirewall firewall delete rule`
✓ **Silent режим** — скрытие консольного окна (флаг `--silent`)
✓ **Dry-run режим** — проверка без реальных изменений (флаг `--dry-run`)
✓ **Целевой инструмент** — удаление только конкретного RAS (флаг `--tool`)
✓ **Детальное логирование** — сохранение всех операций в лог-файл

## Использование

### Запуск через RAST_Deleter.exe (Рекомендуется)

```cmd
REM Запустить самораспаковывающийся архив (автоматически выполнит удаление в silent режиме)
RAST_Deleter.exe
```

Архив автоматически:
1. Распакует содержимое во временную папку
2. Запустит `ras_del.exe --silent -log-dir "%LOCALAPPDATA%\"`
3. Сохранит логи в `%LOCALAPPDATA%\ras_remover_*.log`
4. Очистит временные файлы

### Прямое использование ras_del.exe

#### Базовое использование

```cmd
REM Удалить все обнаруженные RAS (требуются права администратора)
ras_del.exe

REM Проверить что найдётся (dry-run режим — без изменений)
ras_del.exe --dry-run

REM Скрыть консольное окно
ras_del.exe --silent

REM Удалить только конкретный инструмент
ras_del.exe --tool AnyDesk
ras_del.exe --tool TeamViewer
```

#### Расширённые опции

```cmd
REM Сохранить логи в custom папку
ras_del.exe --log-dir C:\MyLogs

REM Комбинировать флаги
ras_del.exe --dry-run --log-dir C:\Logs --silent
ras_del.exe --tool "Chrome Remote Desktop" --dry-run

REM Удалить с логированием в LocalAppData (как RAST_Deleter.exe)
ras_del.exe --silent -log-dir "%LOCALAPPDATA%\"
```

## Архитектура и алгоритм работы

### Общая схема

```
┌─────────────────────────────────────────────────────────────┐
│  1. Проверка прав администратора                            │
│  2. Инициализация логирования                               │
│  3. ФАЗА 1: СКАНИРОВАНИЕ (Detection Phase)                  │
│  4. ФАЗА 2: УДАЛЕНИЕ (Removal Phase)                        │
│  5. ФАЗА 3: ВЕРИФИКАЦИЯ (Verification Phase)                │
│  6. Финальный отчёт                                         │
└─────────────────────────────────────────────────────────────┘
```

### Фаза 1: Сканирование (PHASE 1: SCANNING)

Программа ищет установленные RAS по **четырём независимым источникам**:

#### 1.1 Сканирование процессов
- Перебирает все активные процессы в памяти
- Сравнивает имена процессов с сигнатурами из таблицы `RAS_TOOLS`
- Фиксирует PID и полный путь к исполняемому файлу
- **Результат:** найденные папки установки

#### 1.2 Сканирование реестра (HKLM + HKCU)
- **HKEY_LOCAL_MACHINE:**
  - `HKLM\Software\Microsoft\Windows\CurrentVersion\Uninstall` — глобальные установки
  - Поиск по DisplayName (регистронезависимый)
  - Извлечение `InstallLocation` и `UninstallString`
- **HKEY_CURRENT_USER:**
  - `HKCU\Software\Microsoft\Windows\CurrentVersion\Uninstall` — per-user установки (Chrome RDP, GoTo и т.д.)
- **Результат:** точные пути установки и команды деинсталляции

#### 1.3 Сканирование служб Windows
- Перебирает все Windows-сервисы через WMI/реестр
- Ищет соответствия в списке `service_names` из `RAS_TOOLS`
- **Результат:** список активных служб для удаления

#### 1.4 Сканирование файловой системы
- Проверяет стандартные пути установки:
  - `C:\Program Files\<tool_name>`
  - `C:\Program Files (x86)\<tool_name>`
  - `C:\ProgramData\<tool_name>`
  - Per-user пути в `AppData\Roaming` и `AppData\Local`
- **Результат:** пути даже если реестр повреждён

#### 1.5 Дедупликация путей
- Удаляет дубликаты (case-insensitive)
- Объединяет результаты из всех 4 источников

**Результат ФАЗЫ 1:** Структура `ToolDetectionResult` содержит:
```rust
pub struct ToolDetectionResult {
    tool: &'static RasTool,           // Метаданные инструмента
    found: bool,                       // Обнаружен ли инструмент
    running_processes: Vec<(u32, PathBuf)>,    // PID + пути процессов
    install_dirs: Vec<PathBuf>,                // Папки установки
    uninstall_strings: Vec<String>,            // Команды деинсталляции
    registry_keys_to_delete: Vec<String>,      // Ключи реестра для удаления
    data_dirs: Vec<PathBuf>,                   // Папки данных в AppData
    active_services: Vec<String>,              // Имена служб для удаления
}
```

### Фаза 2: Удаление (PHASE 2: REMOVAL)

Для каждого обнаруженного инструмента выполняется **6-шаговое удаление**:

#### 2.1 Завершение процессов
```
1. Graceful stop (2 секунды ожидания):
   - Запускаем процесс деинсталлятора
   - Даём время корректно завершиться

2. Force kill (если остались процессы):
   - taskkill /F /IM <process_name>
```

#### 2.2 Запуск встроенного деинсталлятора
**КЛЮЧЕВОЕ УЛУЧШЕНИЕ v2.0:**
- Парсим `UninstallString` из реестра
- Извлекаем путь к exe и аргументы
- **Прямой вызов exe без cmd /C:**
  ```rust
  Command::new(executable_path)
      .args(&uninstall_args)
      .output()
  ```
- **Результат:** корректное выполнение без ошибок `'"path"' is not recognized`

#### 2.3 Удаление служб Windows
```
sc delete <service_name>
```
- Удаляет сервисы навсегда из системы
- Требует перезагрузки для полного очищения

#### 2.4 Удаление файлов и папок
- Удаляем `install_dirs` и `data_dirs`
- **Двухуровневая стратегия:**
  1. Используем встроенный Rust `fs::remove_dir_all()`
  2. При ошибке доступа → fallback на `cmd /c rd /s /q`
- **Результат:** папки удаляются даже при блокировке файлов

#### 2.5 Очистка реестра
```
HKLM\Software\Microsoft\Windows\CurrentVersion\Uninstall\<ToolName>
HKCU\Software\Microsoft\Windows\CurrentVersion\Uninstall\<ToolName>
```
- Удаляет ключи через `RegKey::delete_subkey_all()`
- Информация об инструменте исчезает из "Программы и компоненты"

#### 2.6 Очистка файрвола Windows
```
netsh advfirewall firewall delete rule name="<rule_name>"
```
- Находит все правила по имени инструмента
- Удаляет входящие и исходящие правила
- Предотвращает остаточное открытие портов

**Результат ФАЗЫ 2:** Структура `RemovalSummary` содержит статус каждого шага

### Фаза 3: Верификация (PHASE 3: FINAL REPORT)

После удаления программа **повторно сканирует** каждый инструмент:

#### 3.1 Повторное сканирование
- Проверяет процессы (должны быть пусто)
- Проверяет реестр (должны быть пусто)
- Проверяет файловую систему (должны быть пусто)

#### 3.2 Финальный статус
```
✓ OK   — инструмент полностью удален
✗ FAIL — остались следы (требуется ручное вмешательство)
```

#### 3.3 Итоговый отчёт
- Список всех обработанных инструментов
- Статус каждого (удален / остался)
- Код выхода:
  - `0` — все успешно
  - `1` — остались следы

## Технические детали реализации

### Таблица инструментов (RAS_TOOLS)

Программа использует статическую таблицу данных со сведениями о каждом RAS:

```rust
static RAS_TOOLS: &[RasTool] = &[
    RasTool {
        name: "AnyDesk",
        process_names: &["anydesk.exe"],
        service_names: &["AnyDesk"],
        registry_display_names: &["anydesk"],
        exe_filenames: &["AnyDesk.exe"],
        appdata_dirs: &["AnyDesk"],
        firewall_rule_names: &["AnyDesk"],
        silent_args: &["--remove", "--silent"],
    },
    // ... остальные 14 инструментов
];
```

**Преимущества:**
- ✓ Встроены все сигнатуры (нет внешних config файлов)
- ✓ Защита от injection
- ✓ Быстрый поиск по всем критериям
- ✓ Легко добавлять новые инструменты

### Ключевые алгоритмические решения

#### 1. Парсинг UninstallString (Фикс cmd /C бага)

**Проблема (v1.0):**
```
registry: cmd /C "X:\Tools\AnyDesk\AnyDesk.exe" --uninstall --silent
↓
Command::new("cmd /C...")
# ❌ Ошибка: '"X:\Tools\AnyDesk\AnyDesk.exe"' is not recognized
```

**Решение (v2.0):**
```rust
fn parse_uninstall_string(s: &str) -> UninstallerCommand {
    // 1. Извлекаем путь из кавычек или первого токена
    // 2. Парсим аргументы (--uninstall, --silent, /S и т.д.)
    // 3. Возвращаем структуру
    UninstallerCommand {
        executable: PathBuf::from("X:\\Tools\\AnyDesk\\AnyDesk.exe"),
        args: vec!["--uninstall", "--silent"],
    }
}

// Запуск: прямой вызов exe
Command::new(cmd.executable)
    .args(&cmd.args)
    .output()  // ✓ Работает корректно
```

#### 2. Четырёхуровневое сканирование

| Уровень | Ищет | Где | Надёжность |
|---------|------|-----|-----------|
| 1. Процессы | Активные процессы | Память | Только работающие экземпляры |
| 2. Реестр HKLM | InstallLocation, UninstallString | HKLM\...\Uninstall | Почти всегда точно |
| 3. Реестр HKCU | Per-user установки | HKCU\...\Uninstall | Для ChromeRDP, GoTo |
| 4. ФС | Папки по имени | C:\Program Files\*, AppData\ | Backup при повреждении реестра |

**Результат:** Максимальное покрытие, даже если реестр повреждён или установка нестандартная.

#### 3. Двухуровневое удаление файлов

```rust
// Попытка 1: встроенный Rust API (быстро)
match fs::remove_dir_all(&path) {
    Ok(_) => Ok(()),
    Err(_) => {
        // Попытка 2: cmd fallback (медленнее, но пробивает блокировки)
        Command::new("cmd")
            .args(&["/c", "rd", "/s", "/q", &path_str])
            .output()
    }
}
```

**Преимущество:** Удаляет папки даже если файлы заблокированы системой.

#### 4. Case-insensitive дедупликация путей

```rust
fn dedup_paths(paths: Vec<PathBuf>) -> Vec<PathBuf> {
    let mut seen = HashSet::new();
    paths
        .into_iter()
        .filter(|p| seen.insert(p.to_string_lossy().to_lowercase()))
        .collect()
}
```

**Почему нужна:** Windows нечувствительна к регистру в путях → одна папка может быть найдена несколько раз с разными вариантами написания.

## Часто задаваемые вопросы

### Q: Может ли программа удалить RAS, если он запущен?
**A:** Да. Программа сначала пытается gracefully завершить процесс, затем использует force kill (`taskkill /F`), затем удаляет остальное.

### Q: Что делать, если после удаления остались следы?
**A:**
1. Проверьте логи (`--log-dir`) для деталей ошибок
2. Перезагрузитесь (служба Windows может требовать перезагрузку для удаления)
3. Запустите программу снова
4. Если не помогает, проверьте:
   - Права администратора
   - Попытку удаления файлов, занятых другими процессами
   - Заблокированные файлы в реестре

### Q: Отличается ли RAST_Deleter.exe от ras_del.exe?
**A:** Нет, функционально одинаково. Различие:
- `RAST_Deleter.exe` — это SFX архив (Self-Extracting), который распаковывает и запускает `ras_del.exe`
- Удобнее для распространения одним файлом
- Автоматически скрывает консоль и логирует в AppData

### Q: Может ли программа восстановить удаленное ПО?
**A:** Нет. После удаления требуется переустановка из оригинального источника.

### Q: Работает ли программа на Windows 7?
**A:** Нет, требуется Windows 10+. Старые версии имеют другие механизмы реестра и служб.
